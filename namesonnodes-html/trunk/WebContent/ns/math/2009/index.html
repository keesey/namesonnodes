Names on Nodes: MathML Definitions

Version 0.1

T. Michael Keesey

2009 July 11

Abstract.—Mathematical Markup Language 2.0, or MathML 2.0, is an XML language which may be used to model mathematical concepts. Keesey (2007) explored how MathML could be used to model phylogenetic definitions, that is, definitions of taxonomic names which rely on phylogeny (patterns of ancestry and descent), such as those specified in the International Code of Phylogenetic Nomenclature (or PhyloCode). This method involves the use of certain mathematical collections, relations, and operations which relate to phylogeny. In MathML, these mathematical entities may be referenced using a csymbol element which indicates the URL of a definition. For example, the set of all organisms may be referenced as <csymbol definitionURL="http://namesonnodes.org/ns/math/2009#U" xmlns="http://www.w3.org/1998/Math/MathML"/>. This document contains the mathematical definitions needed to model phylogenetic definitions in MathML. These definitions are implemented in the application Names on Nodes.

General Review of Mathematical Terms

Collections

A collection is an entity which consists of zero or more distinct objects. Objects in a collection are members of the collection.

Sets

A set is an unordered collection of elements. When an element, x, is a member of a set, S, this is written x ∈ S. Sets may themselves be members of other sets. Sets may be denoted in the following ways:

  • Extensionally, as a list of elements: {x, y, z}
  • Intensionally, with a rule that determines membership: {x : x > 1}, {x : x has a cellular nucleus}
  • Using a defined name, phrase, or other symbol: ∅, U, Mammalia, YPM-VP 1450

The empty set is the set which includes no members, denoted as ∅.

The cardinality of a set is the number of elements in that set, denoted by surrounding the set with vertical bars, e.g., |S|.

Examples. |∅| = 0. |{1, 2, 3}| = 3.

If all elements of a set, A are members of a set, B, then A is a subset of B, written A ⊆ B. B is a superset of A. Note that all sets are subsets and supersets of themselves. If A ≠ B, then A is a proper subset of B, and B is a proper superset of A: A ⊂ B. Note that ∅ is a subset of all sets.

The following operations may be applied to sets:

  • Union. A ∪ B ≔ {x : x ∈ A or x ∈ B}
  • Intersection. A ∩ B ≔ {x : x ∈ A and x ∈ B}
  • Difference. A − B ≔ {x ∈ A : x ∉ B}

The set consisting of all (relevant) objects is called the universal set, denoted U. The difference of U and a set, S, is the complement of S, written S.

A power set of a set, S, is the set of all subsets of S. The power set of S is written as 2S.

Example. If S = {1, 2}, then 2S = {∅, {1}, {2}, {1, 2}}.

Sequences

A sequence is an ordered collection of elements, generally denoted as a list of elements within parentheses, e.g., (x, y, z). Unlike sets, sequences may have the same member multiple times, e.g., (x, x, y). A sequence with two members is an ordered pair. A sequence with three members is an ordered triple. A sequence with n members is an n-tuple.

A Cartesian product of two factor sets, A and B, is the set of all ordered pairs wherein the first element is a member of A and the second element is a member of B. This is written as A × B. A × A may be written as A2. A Cartesian product may have any number, n, of factor sets, in which case the members of the product are n-tuples.

Example. {1, 2} × {2, 3} = {(1, 2), (1, 3), (2, 2), (2, 3)}.

Relations

A relation is a set of ordered pairs. If (x, y) is a pair in the relation R, this is written as x R y. The first element in such a pair is the predecessor, and the second element is the successor. If x R z and there is no other element, y, such that x R y and y R z, then x is an immediate predecessor of z and z is an immediate successor of x.

A partial order is a relation with the following properties:

  • Reflexivity.—For all x, x ≼ x.
  • Antisymmetry.—For all x and y, if x ≼ y and x ≠ y, then y ⋠ x.
  • Transitivity.—For all x, y, and z, if x ≼ y and y ≼ z, then x ≼ z.

A transitive closure of a relation, R, is the smallest (i.e., least inclusive) transitive relation that includes R.

If ≼ is a partial order and x ≼ y or y ≼ x, then x and y are comparable. If all elements in a set can be compared to each other, then the set is a chain. If no two different elements in a set are comparable, then it is an antichain.

Graphs

A graph is an entity containing a set of objects, called vertices, and connections of these objects, called edges. A graph may be defined as a type of ordered pair, in which the first element is the vertex set and the second element is the edge set. In an undirected graph, each edge is a set of two vertices, indicating that those vertices are connected, or incident. In a directed graph, each edge is an unordered pair of vertices, indicating that the first element, or head, connects to the second element, or tail. Edges in directed graphs may also be called arcs.

A walk in a graph is a sequence of vertices in which each vertex is incident to the adjacent vertices in the sequence. A path is a walk in a directed graph wherein an arc points from one vertex to the next vertex in the sequence. A cycle is a path which begins and ends with the same vertex. A directed graph is said to be acyclic if there are no cycles in it.

Functions

A function maps an element, called an argument, to a value. Formally, a function may be defined as an ordered triple of three sets: f := (X, Y, F). The final set, F, is a set of ordered pairs, wherein the first element is the argument and the second element is the value. There may be only one ordered pair per argument. If an ordered pair, (x, y), is a member of F, this is commonly written as f(x) = y.

The set including all of a function's arguments is the domain. All values of the function are within the target set. The set of all values is the image, which is a subset of the target set. If a function, f, has domain X and codomain Y, this is written as f: X → Y.

General Review of Biological Terms

Organisms

An organism is an individual living entity. Although in many cases it is clear what constitutes an individual, there are notable difficult cases. However, although the mathematical system used by Names on Nodes assumes that there are discrete, individual entities, it never deals with them directly. Thus there is no present need to clarify the term.

Taxonomy

A nonempty set of organisms is a taxon. A taxon whose members are all within another taxon is a subtaxon of that other taxon. A taxon which includes all members of another taxon is a supertaxon of that taxon. The most inclusive taxon is the universal taxon, which includes all organisms. A taxonomy is a system for dividing a taxon into subtaxa.

A taxonomic name is a word or phrase which signifies a taxon. A nomenclatural code is a code of rules which governs taxonomic names.

Specimens

In addition to taxonomic names, taxa may also be referenced using specimens. A specimen is an object which has been catalogued as part of a specimen collection. A specimen collection is often indicated by an abbreviation of its name, specified within the context, e.g., "Yale Peabody Museum: Vertebrate Paleontology Collection" may be abbreviated as YPM-VP. A specimen within a collection may be indicated by the collection's name or abbreviation followed by an identifier that is unique within the collection, e.g., YPM-VP 1450. A specimen may have multiple identifiers if it has been transferred from one collection to another. For example, AMNH 973 and CM 9380 are the same specimen. A specimen may represent no organisms (e.g., a mineralogical specimen), one organism (e.g., a fossil skeleton), or multiple organisms (e.g., a microbe slide).

Character States

Taxa may also be defined intensionally using a description of a necessary characteristic, that is, a character state. Organisms exhibiting the state are part of the taxon. Valid states must be discrete and absolute, that is, organisms cannot partially exhibit them.

Taxa may be defined using a combination of character states. If all states are required for membership, the taxon is monosynthetic. If only one state is required, the taxon is polysynthetic.

Taxonomic names are not usually defined according to character states, but the taxa that the names signify may be diagnosed by character states.

Rank-Based Nomenclature

Taxonomic names may be loosely defined using rank-based definitions. A taxon is rank-defined by specifying one or more type specimens or a type subtaxon (generally, a type), which must be included in the taxon by definition, and a rank, which indicates relative inclusivity of the taxon. Commonly used ranks are, from least to most inclusive, species, genus, family, order, division or phylum, and kingdom. Many others exist as well.

Note that rank-based definitions do not dictate any criteria for membership, apart from the requirement that the type (specimens or subtaxon) must represent included organisms.

A rank-based code is a nomenclatural code which governs rank-based definitions. Currently there are four in effect:

Name of Rank-Based Code Abbreviation Organisms Covered
International Code of Botanical Nomenclature ICBN plants, fungi, some other eukaryotes
International Code of Zoological Nomenclature ICZN animals, some other eukaryotes
International Code of Nomenclature of Prokaryotes ICNP prokaryotes
International Code of Nomenclature for Cultivated Plants ICNCP cultivated plants

Example. Under ICZN rules, the name "Tyrannosaurus rex" refers to a taxon of the species rank typified by CM 9380 (formerly AMNH 973). Therefore, Tyrannosaurus rex must include the organism represented by that specimen. The name "Tyrannosaurus" refers to a taxon of genus rank typified by Tyrannosaurus rex, so it must be a supertaxon of that species (and, by extension, it must include the organism represented by CM 9380). The name "Tyrannosauridae" refers to a taxon of family rank typified by Tyrannosaurus, so it must be a supertaxon of Tyrannosaurus.

Phylogeny

Every organism has one or more ancestors and/or one or more descendants. An immediate ancestor is a parent, and an immediate descendant is a child. The pattern of ancestry and descent among organisms is phylogeny.

Although a fuller correlation wil be made further on, let us note that the biological term "ancestor" correlates to the mathematical term "proper predecessor", and the biological term "descendant" correlates to the mathematical term "proper successor". Therefore, we may say that a predecessor of an organism is any ancestor of that organism, or that organism itself. Conversely, a successor of an organism is any descendant of that organism, or that organism itself. Let us also note that the terms "maximal" and "minimal" may be applied to organisms with regard to taxa. The minimal members of a taxon are those which are not descended from any other member. The maximal members of a taxon are those which are not ancestral to any other member.

The common predecessors of a taxon are all organisms which are predecessors of all members of that taxon. The common successors of a taxon are all organisms which are successors of all members of that taxon.

The exclusive predecessors of a taxon, A, with regard to another taxon, Z, are all common predecessors of A except any which are predecessors of any member of Z.

The comember predecessors of a taxon, A, with regard to another (generally character-based) taxon, M, are all common predecessors of A which are also members of M. A synapomorphic predecessor of a taxon, A, with regard to taxon M, is a comember predecessor for which there is at least one lineage for every member of A satisfying the following conditions:

  1. The first member in the lineage is the synapomorphic predecessor.
  2. The last member in the lineage is the member of A.
  3. All members of the lineage are members of M.

Lineages

A lineage is a sequence of organisms wherein each organism is followed by one of its children.

Cladogens

A taxon which fulfills the following requirements is here termed a cladogen (new term; previously "cladogenetic set" in Keesey [2007]):

  • No member of a cladogen can be ancestral to any other member.
  • There must be at least one organism which is a common successor of all members of the cladogen.

All singleton taxa are cladogens, but cladogens may have larger numbers of members as well.

A node-based cladogen consists of the maximal common predecessors of a taxon. That taxon is a specifying taxon.

A branch-based cladogen consists of the minimal exclusive predecessor of a taxon, the internal specifying taxon, with regard to another taxon, the external specifying taxon.

An apomorphy-based cladogen consists of the minimal synapomorphic predecessor of a taxon, the representative taxon, with regard to another taxon, the apomorphic taxon.

Clades

A clade is taxon including all members of a cladogen and all descendants of all of those members. Clades are monophyletic.

A node-based clade consist of a node-based cladogen and all descendants of all of its members. A branch-based clade consist of a branch-based cladogen and all descendants of all of its members. An apomorphy-based clade consist of an apomorphy-based cladogen and all descendants of all of its members.

Non-Clades

If a taxon includes all members of a cladogen, but not all descendants of those members, then it is paraphyletic. (Note that cladogens themselves are paraphyletic, with the exception of singleton cladogens wherein the single member has no descendants.)

If the maximal common predecessors (i.e., the members of the node-based cladogen) of a taxon are not included within that taxon, then that taxon is polyphyletic.

Phylogeny-Based Nomenclature

A taxon may be strictly defined according to phylogeny, using a phylogeny-based definition. Most commonly, the taxa defined are clades, but other types of taxa may also be phylogenetically defined.

A phylogeny-based code is one which governs phylogeny-based definitions. Currently there are no such codes in effect, but there is a draft of one called the International Code of Phylogenetic Nomenclature (the ICPN or the PhyloCode for short). This code is intended to go into effect in the next few years, exist alognside the rank-based codes, and govern phylogeny-based names for all types of organisms.

Universal Taxon

Definition URL http://namesonnodes.org/ns/math/2009#universalTaxon
Symbol U
Class Set
Definition

U is the set of all organisms, be they extinct, extant, or yet to be.
Implementation org.namesonnodes.math::UniversalSet
Discussion

Although in many cases it is a clear-cut matter as to whether an entity is an organism or not, there are notable difficult cases. However, although the mathematical system used by Names on Nodes assumes that there are discrete entities called "organisms", it never deals with them directly. Rather it deals exclusively with taxa, that is, sets of organisms. Given this, the question of what constitutes an "organism", while interesting in its own right, is not directly relevant to this application. Taxa may be considered subsets of U, without a direct need to specify what the elements are.

Operationally, Names on Nodes treats the smallest discernible taxa in a given case as individuals.

≼[S] includes all members of S and all of each member's ancestors.

This is similar to the concept of "ancestors" except that S ⊆ ≼[S]. "Ancestors" may be thought of as "strict predecessors".

Maximal

Definition URL http://namesonnodes.org/ns/math/2009#max
Symbol max
Class Function
Definition

max : ℘(U) ⟶ ℘(U)
max(S) ≔ {x ∈ S : for all y ∈ S, x ⊀ y}

Implementation org.namesonnodes.math::Maximal
Discussion

The maximal members of a set are all members which are not strictly ancestral to any other member.

The concept of "maximal" correlates to what some authors have termed "last", "latest", or "most recent", as in "most recent common ancestor". However, unlike those other terms, "maximal" is not tied to chronology; the maximal members of a set are not necessarily contemporaries.

Other potential synonyms of "maximal" are "final", "terminal", or "leafmost".

The minimal members of a set are all members which are not strictly descended from any other member.

The concept of "minimal" correlates to what some authors have termed "earliest", "first", or "least recent", as in "least recent common ancestor". However, unlike those other terms, "minimal" is not tied to chronology; the minimal members of a set are not necessarily contemporaries.

Other potential synonyms of "minimal" are "initial" or "rootmost".

Cladogens

Definition URL http://namesonnodes.org/ns/math/2009#cladogens
Symbol Cladogens
Class Set (Finite)
Definition

Cladogens ≔ {S ⊆ U : CS(S) ≠ ∅ and for all x, y ∈ S, x ⊀ y}
Implementation org.namesonnodes.math::Cladogens
Discussion

A cladogen is an independent taxon that has at least one common successor. By "independent", I mean that no member is comparable to (i.e., strictly ancestral to or descended from) any other member.

Note that all singleton sets are cladogens. The empty set is not a cladogen, because CS(∅) = ∅.

If (and only if) S is a cladogen, then ≽[S] is a clade.

Node-Based Predecessors

Definition URL http://namesonnodes.org/ns/math/2009#nodePredecessors
Symbol +
Class Function
Definition

+ : ℘(U) ⟶ ℘(U)
+(S) ≔ max(CP(S))
A + B + ... + Z ≔ +(A ∪ B ∪ ... ∪ Z)

Implementation org.namesonnodes.math::NodePredecessors
Discussion

The node-based predecessors of a set are its maximal common predecessors. This is a similar concept to "most recent common ancestors".

This operation has two forms of notation: 1) as a prefix; and 2) as an infix, which is shorthand for applying the function to a union of sets.

Unless S has no common predecessors, +(S) is a cladogen. Since all known organisms are theorized to descend from common ancestors, the node-based predecessors of all known taxa are cladogens, in theory.

Branch-Based Predecessors

Definition URL http://namesonnodes.org/ns/math/2009#branchPredecessors
Symbol ←
Class Function
Definition

← : ℘(U) × ℘(U) ⟶ ℘(U)
S1 ← S2 ≔ min(CP(S1) − ≼[S2])

Implementation org.namesonnodes.math::BranchPredecessors
Discussion

Specifying branch-based predecessors requires two sets, one internal and one external. The exclusive common predecessors of the internal set are all of its common predecessors minus all predecessors of the external set. The branch-based predecessors are the minimal exclusive common ancestors of the internal set.

Unless S1 has no common predecessors, or all of those common predecessors are also predecessors of S2, then S1 ← S2 is a cladogen.
